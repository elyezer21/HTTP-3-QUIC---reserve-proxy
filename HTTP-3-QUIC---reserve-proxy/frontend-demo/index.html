Este archivo es un ejemplo de referencia para el Equipo 1: muestra cómo el frontend debe hablar con la API REST y el WebSocket a través del Reverse Proxy, sin conocer nunca su IP ni puerto real.

🔑 La regla de oro

El navegador siempre se conecta al mismo host desde donde cargó esta página (el Reverse Proxy). Nunca escribas IPs ni puertos de la API o el WebSocket a mano en el código.

1. Petición a la API REST — fetch('/api/...')

Ruta relativa, sin host: el navegador la envía automáticamente al mismo servidor que sirvió la página. El Caddyfile del proxy intercepta /api/* y la reenvía a la API REST real.

2. Conexión a WebSockets — wss://location.host/ws

Igual que con fetch: se usa location.host para construir la URL dinámicamente. El proxy detecta el "upgrade" de HTTP a WebSocket en la ruta /ws* y lo reenvía al servidor real.

Código fuente de este ejemplo

// ------------------------------------------------------
// 1) API REST: SIEMPRE ruta relativa, nunca IP:puerto fijo
// ------------------------------------------------------
async function probarApi() {
  const out = document.getElementById('apiOut');
  out.textContent = 'Consultando /api/ ...';
  try {
    const res = await fetch('/api/');
    const texto = await res.text();
    out.textContent = texto;
  } catch (e) {
    out.innerHTML = `Error: ${e.message}`;
  }
}

// ------------------------------------------------------
// 2) WebSocket: se arma la URL con location.host,
//    NUNCA con una IP o puerto escrito a mano
// ------------------------------------------------------
let socket = null;

function conectarWs() {
  const out = document.getElementById('wsOut');
  const protocolo = location.protocol === 'https:' ? 'wss://' : 'ws://';
  const url = protocolo + location.host + '/ws?clientId=demo123';

  out.textContent = 'Conectando a ' + url + ' ...';
  socket = new WebSocket(url);

  socket.onopen = () => {
    out.innerHTML += '\nConexión abierta ✔';
  };
  socket.onmessage = (evento) => {
    out.textContent += '\nMensaje recibido: ' + evento.data;
  };
  socket.onerror = () => {
    out.innerHTML += '\nError de conexión (revisa que el backend real de WebSockets esté corriendo)';
  };
  socket.onclose = () => {
    out.textContent += '\nConexión cerrada.';
  };
}

function cerrarWs() {
  if (socket) socket.close();
}
